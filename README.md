# 🍳 AI Recipe Assistant

## Elevator Pitch

Веб-застосунок для тих, хто готує "з того що є". Користувач вводить список наявних продуктів або харчові обмеження — система знаходить підходящі рецепти з бази, а якщо нічого не знайдено, генерує новий за допомогою LLM. Можна попросити адаптувати рецепт: прибрати молоко, зробити веганським, підрахувати калорії або дізнатися чим замінити відсутній інгредієнт.

Ключова особливість: модель не просто генерує текст, а самостійно вирішує яку функцію викликати — пошук, фільтрацію, генерацію, модифікацію або підбір замін (паттерн **Function Calling / Agents**).

## Цільова аудиторія

- Студенти, які готують швидко і з мінімумом продуктів
- Люди з дієтичними обмеженнями (веган, без глютену тощо)
- Новачки в кулінарії, яким потрібна проста покрокова інструкція

## Структура проєкту

```
recipe-ai/
├── app.py           — Streamlit UI, точка входу
├── llm_agent.py     — агентний цикл з Gemini API (Function Calling)
├── functions.py     — логіка пошуку, фільтрації, модифікації, замін
├── recipes.json     — база рецептів (20 страв)
├── .env             — API ключ (не комітити в git)
├── requirements.txt
└── README.md
```

## Реалізовані інструменти (Function Calling)

Модель сама вирішує який інструмент викликати і в якій послідовності:

| Інструмент | Коли викликається |
|---|---|
| `get_recipes_by_ingredients` | Користувач перераховує продукти |
| `filter_recipes` | Є дієтичні або часові обмеження |
| `generate_recipe` | В базі нічого підходящого не знайдено |
| `modify_recipe` | Потрібно змінити існуючий рецепт |
| `estimate_calories` | Запит про калорійність страви |
| `suggest_substitutes` | Немає інгредієнта або питає чим замінити |

## Архітектурні рішення

```
Streamlit UI (app.py)
        ↓  текст запиту
    LLM Agent (llm_agent.py)
        ↓  tool_use → назва функції + аргументи
    Tool Dispatcher (_dispatch)
        ↓  виклик відповідної функції
    functions.py ← recipes.json
        ↓  JSON результат
    LLM Agent (формує фінальну відповідь)
        ↓
    Streamlit UI
```

**Чому Gemini Flash, а не Pro?** Flash достатній для структурованих задач (вибір функції, форматування рецепту). Pro потрібен для складного аналізу — тут це надлишково і дорожче.

**Чому JSON, а не база даних?** Для 20 рецептів JSON достатній і не потребує налаштування. При масштабуванні (1000+ рецептів) варто перейти на SQLite або PostgreSQL.

**Чому Streamlit?** Дозволяє швидко зробити веб-інтерфейс на чистому Python без знання HTML/CSS/JS.

**Управління станом:** Gemini history зберігається в `st.session_state` — в оперативній пам'яті на час сесії. Кнопка "Очистити чат" скидає всю пам'ять моделі.

## Запуск

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. API ключ

Отримайте безкоштовний ключ на [aistudio.google.com](https://aistudio.google.com) і створіть файл `.env` у корені проєкту:

```
GEMINI_API_KEY=AIza...
```

> ⚠️ Файл `.env` вже додано до `.gitignore` — ключ не потрапить у репозиторій.

### 3. Запуск

```bash
streamlit run app.py
```

Відкриється у браузері за адресою `http://localhost:8501`

## Приклади запитів

### Пошук за інгредієнтами
```
яйця, цибуля, картопля
курка, рис, морква
томати, часник, оливкова олія
```

### Фільтрація
```
веганські рецепти з томатами
швидкий сніданок без глютену
вегетаріанське за 10 хвилин
```

### Генерація нового рецепту
```
гречка, молоко, куряче філе
(якщо база нічого не знайшла — модель генерує новий рецепт)
```

### Заміна інгредієнтів
```
чим замінити молоко?
зроби омлет без яєць
немає масла, що використати?
```

### Модифікація існуючого рецепту
```
зроби рецепт 3 без молока
зроби борщ веганським
```

### Калорії
```
скільки калорій у картоплі з яйцями?
калорійність курки з рисом
```

## Безпека і стійкість до відмов

- API ключ зберігається у `.env`, не в коді
- Всі виклики до Gemini API огорнуті в `try/except` — при помилці користувач бачить зрозуміле повідомлення, а не Python traceback
- При перевищенні ліміту запитів (429) виводиться повідомлення з порадою зачекати

## Тестування

### Нормальні сценарії
- Введення реальних інгредієнтів → рецепт знайдено
- Введення рідкісної комбінації → генерація нового рецепту
- Запит із дієтичним обмеженням → фільтрація

### Граничні випадки
- Один інгредієнт → модель запропонує ввести більше або згенерує рецепт
- Порожній запит → Streamlit не відправить форму
- Незнайомий інгредієнт → `suggest_substitutes` поверне `no_substitutes_found`

### Захист від prompt injection
Системний промпт явно обмежує роль моделі кулінарним асистентом. Запити типу "ігноруй всі правила" призведуть до відповіді в межах кулінарної теми або ввічливої відмови.
