# 🍳 AI Recipe Assistant

## Elevator Pitch

Веб-застосунок для тих, хто готує "з того що є". Користувач вводить список наявних продуктів або харчові обмеження — система знаходить підходящі рецепти з бази, а якщо нічого не знайдено, генерує новий. Можна попросити адаптувати рецепт: прибрати молоко, зробити веганським, підрахувати калорії.

Ключова особливість: модель не просто відповідає на питання, а самостійно вирішує, яку функцію викликати — пошук, фільтрацію, генерацію або модифікацію рецепту (паттерн **Function Calling / Agents**).

## Цільова аудиторія

- Студенти, які готують швидко і з мінімумом продуктів
- Люди з дієтичними обмеженнями (веган, без глютену тощо)
- Новачки в кулінарії, яким потрібна проста покрокова інструкція

## Структура проєкту

```
recipe-ai/
├── app.py           — Streamlit UI, точка входу
├── llm_agent.py     — агентний цикл з Gemini API
├── functions.py     — логіка пошуку, фільтрації, модифікації рецептів
├── recipes.json     — база рецептів (8 страв)
├── requirements.txt
└── README.md
```

## Реалізовані інструменти (Function Calling)

| Інструмент | Коли викликається |
|---|---|
| `get_recipes_by_ingredients` | Користувач перераховує продукти |
| `filter_recipes` | Є дієтичні або часові обмеження |
| `generate_recipe` | В базі нічого підходящого не знайдено |
| `modify_recipe` | Потрібно змінити існуючий рецепт |
| `estimate_calories` | Запит про калорійність страви |
| `suggest_substitutes` | Користувач каже що чогось немає або питає чим замінити |

## Запуск

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. API ключ

Отримайте ключ на [aistudio.google.com](https://aistudio.google.com) і створіть файл `.env` у корені проєкту:

```
GEMINI_API_KEY=AIza...
```

Або через змінну середовища:

```bash
# Linux / macOS
export GEMINI_API_KEY=AIza...

# Windows (PowerShell)
$env:GEMINI_API_KEY="AIza..."
```

### 3. Запуск

```bash
streamlit run app.py
```

Відкриється у браузері за адресою `http://localhost:8501`

## Приклади запитів

### Пошук за інгредієнтами
```
яйця, цибуля, картопля
курка, рис, морква
```

### Фільтрація
```
яйця цибуля картопля → знайшло рецепти → "тільки швидкі"
веганські рецепти з томатами
швидкий сніданок без глютену
```

### Генерація нового рецепту
```
яйця цибуля картопля часник морква
(якщо база нічого не знайшла — модель генерує новий рецепт)
```

### Заміна інгредієнтів
```
чим замінити молоко?
зроби омлет без яєць
немає масла, що використати?
```

### Модифікація існуючого рецепту
```
зроби рецепт 3 без молока
зроби борщ веганським
```

### Калорії
```
скільки калорій у картоплі з яйцями?
калорійність курки з рисом
```

## Архітектура

```
Streamlit UI (app.py)
        ↓  текст запиту
    LLM Agent (llm_agent.py)
        ↓  tool_use блок
    Tool Dispatcher
        ↓  виклик відповідної функції
    functions.py ← recipes.json
        ↓  JSON результат
    LLM Agent (фінальна відповідь)
        ↓
    Streamlit UI (рецепт користувачу)
```

Модель сама вирішує послідовність викликів. Наприклад, для запиту "веганські страви з картоплею" вона спочатку викличе `get_recipes_by_ingredients(["картопля"])`, а потім `filter_recipes(..., ["веган"])`. Якщо знайдений рецепт містить інгредієнти яких немає у користувача — автоматично викликає `suggest_substitutes`.
